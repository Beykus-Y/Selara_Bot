# Видимые противоречия

Этот файл фиксирует противоречия, которые были обнаружены между документацией, конфигурацией и фактическим состоянием проекта.

## 1) Историческое расхождение: список команд в README и реальные команды бота

### Наблюдение
Ранее в README перечислялась только часть команд, тогда как в runtime зарегистрирован более широкий набор (включая `/article`, `/craft`, `/auction`, `/bid`, `/title`, `/adopt`, `/pet`, `/family`, `/settrigger`, `/rpadd` и другие).

### Противоречие
Публичная документация отставала от фактического API команд.

---

## 2) Историческое расхождение: ссылки на workflow были в локальном формате

### Наблюдение
В README встречались пути вида `/mnt/c/...`, привязанные к локальной машине.

### Противоречие
Документация репозитория должна использовать относительные проектные ссылки, а не machine-specific пути.

---

## 3) Команда `pytest` в отрыве от требований окружения

### Наблюдение
Указание «просто запустите `pytest`» противоречило реальности: без `.[dev]` тесты не собираются.

### Противоречие
Инструкция была неполной относительно обязательных предварительных шагов.

---

## 4) Quick start и скрытая зависимость от внешней Docker-сети

### Наблюдение
`docker-compose.yml` ожидает существование внешней сети `edge`, но в «быстром запуске» это часто не подчёркнуто.

### Противоречие
Формально описан простой старт, но фактически есть дополнительное инфраструктурное условие.
