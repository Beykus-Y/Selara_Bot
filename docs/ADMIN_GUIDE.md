# Selara — руководство для админов групп

Этот документ предназначен для администраторов Telegram‑групп, которые добавляют бота Selara в чат.

## 1. Коротко о боте
Selara — бот для статистики, игр, экономики и модерации. Он умеет:
- считать активность и строить топы
- вести репутацию и карму
- запускать мини‑игры
- давать экономику (ферма, клики, лотерея, рынок)
- модерировать (преды, варны, баны)
- работать с ролями внутри бота

## 2. Подготовка перед добавлением
Перед добавлением в группу проверьте:
- У вас есть права администратора в группе.
- Вы знаете точный `@username` бота (например, `@SelaraBot`).
- Если нужны текстовые команды без `/` и реакции на обычные сообщения (карма `+/-`, социальные действия), необходимо отключить privacy mode в BotFather.

### Отключение privacy mode (важно для текстовых команд)
1. Откройте BotFather.
2. Команда `/setprivacy`.
3. Выберите бота.
4. Установите `Disable`.

Без этого бот будет получать только команды и упоминания, и часть функций не будет работать.

## 3. Добавление бота в группу
1. В группе нажмите «Добавить участника».
2. Введите `@username` бота.
3. Добавьте бота в группу.
4. (Рекомендуется) сразу сделайте бота администратором.

## 4. Права администратора бота
Минимальные права для базовой работы:
- Отправка сообщений
- Отправка медиа

Для модерации боту нужны дополнительные права:
- Блокировка пользователей (Ban users / Restrict users)

Если бота не сделать администратором, команды модерации будут работать только «внутренне» (статистика будет обновляться), но фактический Telegram‑бан не применится.

## 5. Инициализация ролей бота
Внутри Selara есть собственные роли:
- `Участник`
- `Мл. админ`
- `Старший админ`
- `Совладелец`
- `Владелец`

Также можно создавать кастомные роли (например, `Видеомонтажер`) из шаблона и отдельно редактировать их права/ранг.

### Первый владелец
Если в чате ещё нет ролей, первый вызов явной role-команды (`/roles`, `/roleadd`, `/roleremove`, `/rolecreate`, `/roleperms` и т.д.) создаёт первого `Владельца` автоматически.

### Основные команды ролей
- `/roles` — список ролей
- `/roledefs` — список всех доступных ролей (system/custom) и прав
- `/roleadd "<role>" @user` — назначить роль
- `/roleremove @user` — снять роль
- `/roletemplates` — показать системные шаблоны ролей
- `/rolecreate "<название>" ["шаблон"] [ранг]` — создать кастомную роль
- `/rolesettitle "<роль>" "<новое название>"` — переименовать кастомную роль
- `/rolesetrank "<роль>" <ранг>` — изменить ранг кастомной роли
- `/roleperms "<роль>" +право -право ...` — редактировать права кастомной роли
- `/roledelete "<роль>"` — удалить кастомную роль

Пример:
- `/roleadd "Совладелец" @username`
- `/rolecreate "Видеомонтажер" "Мл. админ" 15`
- `/roleperms "Видеомонтажер" +announce -manage_games`
- `/roleremove @username`

Через ЛС-панель (`/start` -> группа -> `Роли`) можно делать то же самое текстовыми командами.

## 6. Настройки группы
Команды:
- `/settings` — показать текущие настройки
- `/setcfg <key> <value>` — изменить настройку (нужно право `manage_settings`)
- `/setrank "<команда>" "<роль>"` — установить минимальный ранг для команды
- `/ranks` — показать все настроенные ранги команд
- Текстом в группе: `установить "команда" ранг внутри бота роль`
- Текстом в ЛС-панели (`/start` -> группа -> `Ранги команд`) работают те же форматы.

Сброс к значениям по умолчанию (из env):
- `/setcfg <key> default`

### Список ключей настроек
Ниже перечень ключей и краткое описание. Точные текущие значения и дефолты смотрите в `/settings`.

**Лимиты и топы**
- `top_limit_default` — размер топа по умолчанию
- `top_limit_max` — максимальный размер топа
- `vote_daily_limit` — дневной лимит голосов кармы
- `leaderboard_hybrid_karma_weight` — вес кармы в гибридном топе (0..1)
- `leaderboard_hybrid_activity_weight` — вес активности в гибридном топе (0..1)
- `leaderboard_7d_days` — количество дней для топа 7d
- `leaderboard_week_start_weekday` — день старта недели для недельного топа (0=пн ... 6=вс)
- `leaderboard_week_start_hour` — час старта недели в UTC (0..23)

**Мафия**
- `mafia_night_seconds` — длительность ночи
- `mafia_day_seconds` — длительность дня
- `mafia_vote_seconds` — длительность голосования
- `mafia_reveal_eliminated_role` — показывать ли роль выбывшего (true/false)

**Текстовые команды и контент**
- `text_commands_enabled` — включить текстовые команды (true/false)
- `text_commands_locale` — язык текстовых команд (`ru` или `en`)
- `actions_18_enabled` — включить встроенные 18+ reply-действия, «дрочку» и 18+ темы для `Кто я` (true/false)

**Экономика**
- `economy_enabled` — включить экономику (true/false)
- `economy_mode` — режим экономики (`global` или `local`)
- `economy_tap_cooldown_seconds` — кулдаун на `/tap`
- `economy_daily_base_reward` — базовая награда `/daily`
- `economy_daily_streak_cap` — максимум стрика `/daily`
- `economy_lottery_ticket_price` — цена билета лотереи
- `economy_lottery_paid_daily_limit` — лимит платных билетов в день
- `economy_transfer_daily_limit` — дневной лимит переводов
- `economy_transfer_tax_percent` — налог на переводы (0..100)
- `economy_market_fee_percent` — комиссия рынка (0..100)
- `economy_negative_event_chance_percent` — шанс негативного события (0..100)
- `economy_negative_event_loss_percent` — процент потерь при негативном событии (0..100)

## 7. Модерация
Команды модерации работают в группе и требуют право `moderate_users`.

Команды:
- `/pred` — выдать предупреждение
- `/warn` — выдать варн
- `/unwarn` — снять варн
- `/ban` — забанить
- `/unban` — разбанить
- `/modstat` — посмотреть статус предов/варнов/банов

Можно использовать reply‑команды без `/`:
- `пред`, `варн`, `снять варн`, `бан`, `разбан`, `повысить`, `понизить`

Правило:
- `3 преда = 1 варн`
- `3 варна = 1 бан`

Если у бота нет права «Блокировать пользователей», Telegram‑бан не применится, но внутренний статус обновится.

## 8. Кастомные алиасы текстовых команд
Админ с правом `manage_settings` внутри бота может задать свои триггеры для текстовых команд в конкретной группе.

Команды:
- `/setalias "стандартный триггер" "новый алиас" [--force]` — добавить или переназначить алиас
- `/aliases` — показать текущий режим и список алиасов
- `/unalias "алиас"` — удалить алиас
- `/aliasmode aliases_if_exists|both|standard_only` — режим применения алиасов

Режимы:
- `both` — работают и стандартные, и кастомные триггеры
- `aliases_if_exists` — если у команды есть кастомный алиас, стандартный триггер этой команды не работает
- `standard_only` — работают только стандартные триггеры

Примеры:
- `/setalias "нейминг" "+ник"`
- `/setalias "кто я" "мой профиль"`
- `/setalias "нейминг" "+ник" --force`
- `/aliasmode aliases_if_exists`

Важно:
- Встроенные стандартные триггеры защищены от перехвата.
- `--force` нужен только для конфликта с уже существующим кастомным алиасом.

## 9. Объявления
Команда:
- `объява "текст"` — пинг участников и 5 сообщений объявления (нужно право `announce` или подходящий ранг команды)

Участники могут отключать теги объявления:
- `анрег` — выключить теги
- `рег` — включить теги

## 9.1. Смарт-триггеры и шаблонные переменные
Смарт-триггеры можно создавать как из Telegram, так и из web-панели группы.

Команды:
- `/settrigger "ключ" "ответ" [exact|contains|starts_with]`
- `/triggers` — список текущих смарт-триггеров
- `/deltrigger <id>` — удалить смарт-триггер
- `/triggervars` — показать все доступные `{переменные}`

Кастомные RP-действия используют тот же шаблонизатор:
- `/rpadd "триггер" "шаблон"`
- `/rps`
- `/rpdel "триггер"`

Полезные переменные:
- `{user}` / `{actor}` — отправитель как упоминание
- `{user_name}` — имя отправителя без ссылки
- `{reply_user}` / `{target}` — пользователь из reply
- `{reply_text}` — текст reply-сообщения
- `{chat}` — название чата
- `{text}` — текст текущего сообщения
- `{args}` — хвост после ключевой фразы
- `{date}`, `{time}`, `{datetime}`, `{weekday}` — дата и время по UTC

Примеры:
- `/settrigger "кто тут" "Сейчас тут {user}, чат: {chat}, время: {time}"`
- `/settrigger "привет" "Привет, {user_name}. Ты ответил(а) на: {reply_text}"`
- `/rpadd "куснуть" "{actor} кусает {target}. Комментарий: {args}"`

Важно:
- Для буквальных фигурных скобок используйте `{{` и `}}`.
- Reply-переменные работают только если исходное сообщение было отправлено reply.
- В web-панели полный список переменных доступен на странице группы и в `/app/docs/admin`.

## 10. Игры и ЛС
Некоторые игры требуют личных сообщений с ботом (роли, ответы, приватные карточки).

Рекомендуйте участникам:
- открыть чат с ботом и нажать `Start`
- разрешить боту писать в ЛС

Без этого бот не сможет отправить роли и приватные подсказки.

## 11. Проверка базовой работы
После добавления сделайте быстрый чек:
1. `/help` — бот отвечает
2. `/me` — строит профиль
3. `/game` — открывает меню игр
4. `/settings` — показывает настройки

## 12. Быстрые проблемы и решения
- Бот не реагирует на текстовые команды: включите `text_commands_enabled` и отключите privacy mode в BotFather.
- Команда `/ban` пишет, что бан не применён: дайте боту права на блокировку.
- Участник не получает роль в ЛС: пусть откроет диалог с ботом и нажмёт `Start`.
- Команды настроек/рангов не работают: убедитесь, что у вашей роли есть нужные права (`manage_settings` или `manage_command_access`).
